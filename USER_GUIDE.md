# DBCLI User Guide

## 🚀 How to Use the Database Migration Tool

### Prerequisites

#### System Requirements
- **Operating System**: Windows, Linux, or macOS
- **Go Runtime**: Go 1.19+ (if running from source)
- **Memory**: 128MB minimum, 512MB recommended
- **Storage**: 50MB for application + space for logs

#### Database Requirements
- **MySQL**: Version 5.7+ or 8.0+
- **Oracle**: Version 11g+ (with appropriate drivers)
- **Network Access**: Connectivity to both source and destination databases
- **Database Permissions**: Read access to source, Write access to destination

### Installation & Setup

#### Option 1: Using Pre-built Executable
```bash
# Download the dbcli.exe file to your desired directory
# No additional installation required
```

#### Option 2: Building from Source
```bash
git clone https://github.com/pclubiitk/dbcli.git
cd dbcli
go build -o dbcli.exe
```

### Quick Start Guide

#### 1. Basic Usage
```bash
# Run the application
./dbcli.exe

# Or on Windows
dbcli.exe
```

#### 2. First-Time Setup
When you run DBCLI for the first time, it will:
- Create a default configuration file (`config.yaml`)
- Set up logging directory
- Initialize all components

### Step-by-Step Migration Process

#### Step 1: Source Database Credentials
When DBCLI starts, you'll be prompted to enter source database details:

```
Source Database Configuration:
- Database Vendor: [mysql/oracle]
- Host: [localhost or IP address]
- Port: [3306 for MySQL, 1521 for Oracle]
- Username: [your database username]
- Password: [your database password]
- Database: [database/schema name]
```

**Example for MySQL:**
```
Database Vendor: mysql
Host: localhost
Port: 3306
Username: root
Password: mypassword
Database: source_db
```

**Example for Oracle:**
```
Database Vendor: oracle
Host: localhost
Port: 1521
Username: hr
Password: oracle
Database: XE
```

#### Step 2: Destination Database Credentials
After source configuration, enter destination database details using the same format.

#### Step 3: Source Table Selection
- DBCLI will display all available tables from the source database
- Use arrow keys to navigate
- Press Enter to select a table
- Press 'q' to quit or 'h' for help

#### Step 4: Source Column Selection
- View all columns in the selected source table
- Use Space to select/deselect columns
- Use arrow keys to navigate
- Press Enter when done selecting
- You can select multiple columns for migration

#### Step 5: Destination Table Selection
- Choose the destination table from available tables
- Or specify a new table name (DBCLI will create it)

#### Step 6: Destination Column Selection
- Select which columns in the destination table to map to
- Must match the number of selected source columns

#### Step 7: Column Mapping
- Map each source column to a destination column
- DBCLI will suggest automatic mappings based on column names
- Type the destination column name for each source column
- Press Enter to confirm each mapping

#### Step 8: Dump Options
- Choose whether to generate a SQL dump file
- Specify the path for the dump file (optional)
- This creates a backup of your migration commands

#### Step 9: Migration Confirmation
- Review all your selections
- Source and destination details
- Column mappings
- Migration settings
- Press 'y' to proceed or 'n' to cancel

### Advanced Features

#### Configuration File (config.yaml)
DBCLI creates a configuration file with default settings:

```yaml
database:
  defaultVendor: "mysql"
  defaultHost: "localhost"
  defaultPorts:
    mysql: "3306"
    oracle: "1521"
  connectionPool:
    maxConnections: 10
    maxIdleTime: 300
    connMaxLifetime: 3600
  connectionTimeout: 30

migration:
  defaultBatchSize: 1000
  enableTransactions: true
  enableProgress: true
  autoGenerateDump: false
  defaultDumpPath: "./dumps"
  enableCheckpoints: true
  validationRules:
    checkDataTypes: true
    checkNullConstraints: true
    checkDataIntegrity: true
  commonMappings:
    id: "id"
    name: "name"
    email: "email"

logging:
  level: "info"
  enableFileOutput: true
  logFilePath: "./logs/dbcli.log"
  maxFileSize: 100
  maxBackups: 5
  enableJSON: false
  enableConsole: true

ui:
  showProgress: true
  enableHelp: true
  confirmationRequired: true
  theme: "default"
  enableColors: true
```

#### Help System
Press 'h' at any step to get context-specific help:
- Navigation instructions
- Available shortcuts
- Step-specific guidance
- Troubleshooting tips

#### Checkpoint/Resume Feature
- DBCLI automatically saves progress during large migrations
- If migration is interrupted, you can resume from the last checkpoint
- Checkpoints are saved in `./checkpoints/` directory

#### Validation Features
- Pre-migration validation checks data compatibility
- Verifies data types match between source and destination
- Checks for NULL constraint violations
- Validates data integrity before migration starts

#### Dry-Run Mode
- Preview migrations without executing them
- See what data would be migrated
- Identify potential issues before actual migration
- Generate migration reports

### Keyboard Shortcuts

#### Navigation
- **Arrow Keys**: Navigate through lists and options
- **Enter**: Select/confirm current item
- **Space**: Toggle selection (for multi-select lists)
- **Tab**: Move to next input field
- **Shift+Tab**: Move to previous input field

#### Global Shortcuts
- **h**: Show help for current step
- **q**: Quit application (with confirmation)
- **Ctrl+C**: Emergency exit
- **b**: Go back to previous step
- **r**: Reset current step

#### Migration Control
- **p**: Pause migration (if supported)
- **s**: Show migration statistics
- **l**: View logs in real-time

### Common Use Cases

#### 1. Simple Table Copy
```
Source: MySQL table 'users' 
Destination: MySQL table 'users_backup'
Columns: All columns (1:1 mapping)
```

#### 2. Cross-Database Migration
```
Source: Oracle table 'EMPLOYEES'
Destination: MySQL table 'employees'
Columns: Selected columns with name transformation
```

#### 3. Data Transformation
```
Source: MySQL table 'old_customers'
Destination: MySQL table 'customers'
Columns: Rename and restructure columns during migration
```

#### 4. Partial Data Migration
```
Source: Large table with millions of rows
Destination: Filtered subset based on selected columns
Strategy: Use checkpoints for resumable migration
```

### Troubleshooting

#### Common Issues

**1. Connection Failed**
- Verify database credentials
- Check network connectivity
- Ensure database server is running
- Verify firewall settings

**2. Permission Denied**
- Check user permissions on source database (SELECT)
- Check user permissions on destination database (INSERT, CREATE)
- Verify schema access rights

**3. Data Type Mismatch**
- Use the validation feature to identify incompatible types
- Review column mappings
- Consider data transformation needs

**4. Migration Interrupted**
- Use checkpoint files to resume
- Check logs for error details
- Verify destination database state

**5. Performance Issues**
- Adjust batch size in configuration
- Check connection pool settings
- Monitor system resources

#### Log Files
Check logs for detailed error information:
- **Location**: `./logs/dbcli.log`
- **Rotation**: Automatic rotation when files get large
- **Levels**: DEBUG, INFO, WARN, ERROR

#### Getting Help
- Use 'h' key for context-sensitive help
- Check log files for error details
- Review configuration settings
- Verify database connectivity

### Best Practices

#### Before Migration
1. **Backup** your destination database
2. **Test** with a small dataset first
3. **Verify** source and destination schemas
4. **Check** available disk space
5. **Plan** for downtime if needed

#### During Migration
1. **Monitor** progress through the UI
2. **Check** logs for any warnings
3. **Verify** data integrity periodically
4. **Keep** network connection stable

#### After Migration
1. **Verify** data completeness
2. **Check** data integrity
3. **Update** application connections
4. **Archive** migration logs
5. **Clean up** temporary files
